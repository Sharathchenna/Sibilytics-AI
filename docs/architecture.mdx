## High-Level Architecture

Sibilytics AI follows a modern client-server architecture with a React-based frontend and a Python backend.

## Data Flow

<Steps>
  <Step title="Upload">
    Signal files are uploaded through the web interface with progress tracking and gzip compression support.
  </Step>
  <Step title="Parse">
    Files are parsed with auto-detection of headers, delimiters, and data types.
  </Step>
  <Step title="Cache">
    Parsed data is cached on the server to avoid re-uploading for subsequent operations.
  </Step>
  <Step title="Process">
    Wavelet decomposition is applied for signal denoising with configurable parameters.
  </Step>
  <Step title="Extract">
    Statistical features (mean, std, skewness, entropy, etc.) are extracted from signals.
  </Step>
  <Step title="Visualize">
    Interactive plots are generated using Plotly.js for time-domain, frequency-domain, and spectrogram analysis.
  </Step>
</Steps>

## Backend Structure

```
backend/
├── main.py              # FastAPI app, signal processing, SVM
├── ann_router.py        # ANN endpoints
├── data_viz.py          # Data visualization endpoints
├── requirements.txt     # Python dependencies
├── Dockerfile           # Container configuration
└── docker-compose.yml   # Production deployment
```

## Frontend Structure

```
frontend/app/
├── page.tsx                    # Main landing page
├── layout.tsx                  # Root layout
├── globals.css                 # Global styles
├── components/
│   ├── SignalProcessor.tsx     # Signal processing UI
│   ├── SVMClassifier.tsx       # ML module (SVM + ANN)

│   ├── PlotDisplay.tsx         # Plot rendering

└── lib/
    ├── api.ts                  # API functions
    ├── ann-api.ts              # ANN-specific API
    └── plotly-theme.ts         # Plotly configuration
```
